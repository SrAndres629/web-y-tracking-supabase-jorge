## 2026-04-24 - Icon-Only Buttons Missing Context and Keyboard Focus
**Learning:** Found a recurring accessibility issue where icon-only buttons (like social media links in footers and close buttons in mobile menus) lack `aria-label` attributes and keyboard focus indicators (`focus-visible`). This makes these interactive elements essentially invisible or difficult to use for screen reader users and those navigating via keyboard. Purely visual elements like icons (`<i>`) or typography entities (`&times;`) also lack `aria-hidden="true"`.
**Action:** When auditing UI components, always ensure that any icon-only interactive element has an `aria-label` (describing its purpose, e.g., "Facebook" or "Cerrar menú"), applies `aria-hidden="true"` to its internal visual content, and implements clear focus styles (e.g., `focus-visible:ring-2`) to ensure full keyboard navigation support.

## 2025-04-23 - Focus Rings and ARIA Context in Template Components
**Learning:** Found a pattern across the application's components (`navbar.html`, `footer.html`) where highly stylized icon-only interactive elements (such as hamburger menus, close buttons, and social media links) lacked explicit `aria-label`s. More critically, keyboard users were unable to discern navigation flow because custom styling stripped standard focus indicators without replacing them (no `focus-visible` ring).
**Action:** Always verify that purely visual elements (`<i class="...">`, `&times;`) inside buttons/links have `aria-hidden="true"`, that the parent interactive element has a descriptive `aria-label`, and use Tailwind's `focus-visible:outline-none focus-visible:ring-2` to restore keyboard accessibility while preserving mouse interaction UX.

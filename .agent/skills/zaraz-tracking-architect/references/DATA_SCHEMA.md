# Tracking Data Schema: Jorge Aguirre Flores

## Standard Event Properties
Every event sent to Zaraz/CAPI should include:
- `event_name`: (String) e.g., 'Contact', 'Lead', 'ViewContent'.
- `event_id`: (String) Unique ID for deduplication.
- `event_source_url`: (String) Full URL where the event occurred.
- `action_source`: (String) Default 'website'.

## User Data (Normalized)
PII data must be hashed (SHA256) before reaching Meta CAPI.
- `em`: Email (lowercase, trimmed).
- `ph`: Phone (digits only, including country code).
- `external_id`: Persistent UUID generated by the system.

## Custom Data
- `content_name`: Name of the service/page.
- `content_category`: Category of the interaction.
- `value`: (Optional) Monetary value.
- `currency`: Default 'USD' or 'BOB'.
